## 1. 비주얼 테마 & 분위기

딥노이드의 모든 결과물은 **명확한 메시지 전달**을 최우선으로 합니다. 무언가를 만들기 전에 항상 두 가지를 먼저 생각하세요 — 어떤 메시지를 전달할 것인가, 어떤 흐름으로 구성할 것인가. 
흐름이 결정되기 전에 제작을 시작하지 마세요.

**Less is More**: 한 화면(슬라이드, 섹션, 페이지)에는 한 가지 메시지면 충분합니다. 발표용이라면 스피치를 직관적으로 보여주는 데 집중하고, 자료제출형이라면 텍스트가 많아도 괜찮습니다. 본문 내용은 항상 상단의 Head Message를 뒷받침합니다.

**제작 순서**: 스토리라인 구성 → 고스트 덱(Ghost Deck) 초안 작성 → 자료·데이터 추가 → 제작(Production). 고스트 덱은 손그림, Post-it, iPad 어느 방식이든 가능합니다.

**Answer First (Message First)**: 스토리라인은 반드시 **결론(헤드 메시지)부터** 세웁니다. 각 페이지의 헤드 메시지(제목)를 먼저 확정하되 누구나 이해할 수 있는 **쉬운 표현**으로 쓰고, 본문은 그 헤드 메시지를 뒷받침하도록 구성합니다. 헤드 메시지가 정해지기 전에는 세부 제작에 들어가지 않습니다.

### 핵심 특성

이 디자인 시스템을 한눈에 요약합니다. 세부 규칙에 앞서 아래 원칙을 먼저 숙지하세요.

- **메시지 우선 · Less is More** — 한 화면에는 한 가지 메시지. 본문은 상단 Head Message를 뒷받침
- **내지는 `{colors.white}` 흰 배경 고정** — 장식용 바·사이드바·밑줄 등 일절 금지
- **메인 컬러는 BLUE·GREEN 두 색** — `{colors.primary}` DEEPNOID BLUE와 `{colors.green}` DEEPNOID GREEN **둘 다 메인**. BLUE 한 색만 쓰지 말고 두 색을 함께 사용
- **BLUE→GREEN 그라데이션 적극 활용** — 두 메인 색을 잇는 그라데이션이 딥노이드 시그니처 그래픽
- **공식 팔레트 8색**(메인 2 + 보조 6) — 보조 6색은 역할이 고정되어 있어 자유 강조용이 아님
- **Pretendard 단일 서체** — ExtraBold / Bold / Regular 3가지 Weight만 사용
- **자간(letter-spacing)은 보통(0) 고정** — 좁히지도, **넓히지도** 말 것
- **둥근 모서리 기본**(`{rounded.shape}` 10px / 캡슐형 `{rounded.capsule}` 50%) — 텍스트·표·단순 도형 중심 구성
- **정렬은 왼쪽 또는 가운데만** · 기준 여백 준수 · 1–4단 그리드 혼합 금지
- **내지 본문은 §6 패턴 5종**(카드 / 카드그리드 / 비교형 / 스텝형 / KPI형) **중 1종**으로만 구성 — 같은 패턴 연속 사용 금지

| 페이지 유형 | 기조 |
|---|---|
| **표지 (Cover)** | 브랜드 아이덴티티 표현. `{colors.primary}` 기반 도형·곡선 요소 사용 가능 |

## 2. 색상 팔레트 & 역할

딥노이드의 공식 팔레트는 **메인 2색 + 보조 6색, 총 8색**입니다. 메인 컬러 `{colors.primary}` BLUE와 `{colors.green}` GREEN은 **둘 다 적극적으로 사용**하며, BLUE 한 색만 쓰지 않습니다. 팔레트 외 색상은 임의로 추가하지 않습니다.

> **AI 에이전트 필독**: 메인 컬러는 `{colors.primary}` BLUE와 `{colors.green}` GREEN **두 색입니다 — 두 색을 모두 사용해야 하며, BLUE 한 색만 쓰는 것은 금지입니다.** 한편 팔레트에 8색이 있다는 것이 "8색을 모두 쓰라"는 뜻은 **아닙니다** — 보조 6색은 역할이 고정되어 해당 조건에서만 등장합니다. 강조는 메인 2색(BLUE·GREEN) 위주로 구성하세요.

### 메인 컬러 (2색)

| 토큰 | HEX | 역할 |
|---|---|---|
| `{colors.primary}` | `#0066FF` | 브랜드 대표색·주조색. 표지 도형, CTA, 핵심 강조 |
| `{colors.green}` | `#34D0B3` | DEEPNOID GREEN. BLUE와 동등한 **공동 메인** — 의료 한정이 아니라 전 분야 강조·구성에 사용 |

### 보조 컬러 (6색) — 역할 고정

보조 6색은 **자유롭게 쓰는 강조 팔레트가 아닙니다.** 아래 정해진 조건에서만 등장합니다.

| 토큰 | HEX | 분류 | 사용 조건 |
|---|---|---|---|
| `{colors.white}` | `#FFFFFF` | 기반 | 기본 배경 (내지 고정) — 항상 |
| `{colors.black}` | `#000000` | 기반 | 기본 텍스트 — 항상 |
| `{colors.light-green}` | `#D6F6F0` | 배경 틴트 | 영역 구분용 옅은 배경에만. 텍스트·도형 채우기 금지 |
| `{colors.light-blue}` | `#CCE0FF` | 배경 틴트 | 영역 구분용 옅은 배경에만. 텍스트·도형 채우기 금지 |
| `{colors.navy}` | `#000057` | 기능 | 비교 표현(대조 항목)에만 |
| `{colors.red}` | `#FF4242` | 기능 | 경고·오류·음수 지표에만 |
| `{colors.SECURITY-green}` `#00C857` | 기능 | 산업, 보안 본부에만 사용 |

### 한 화면당 색상 사용 한도

PPT 추출 시 보조색이 과다 유입되는 것을 막기 위해, 한 화면(슬라이드·페이지)에서의 색상 수를 아래로 제한합니다.

| 목적 | 허용 한도 |
|---|---|
| 강조 | 메인 2색 `{colors.primary}` BLUE·`{colors.green}` GREEN 사용 (+ 필요 시 보조 1색) |
| 배경 틴트 | `{colors.light-green}` / `{colors.light-blue}` 중 1색 |
| 기능색 | `{colors.navy}`·`{colors.red}`는 해당 조건 충족 시에만 등장 |

### 그라데이션

두 메인 컬러를 잇는 **BLUE → GREEN 그라데이션**은 딥노이드의 **시그니처 그래픽 표현**입니다. 표지·핵심 강조 요소·도형 채우기 등에 **적극적으로 활용**합니다. (내지 본문 배경, 간지 배경에는 사용하지 않습니다.)

| 종류 | 구성 | 사용 |
|---|---|---|
| **메인 그라데이션** (의료 그라데이션) | `{colors.primary}` BLUE → `{colors.green}` GREEN | 딥노이드 대표 그라데이션 — 표지·핵심 강조 등 전반에 **적극 활용** (간지 배경에는 사용하지 않음) |
| 산업 그라데이션 | `{colors.primary}` BLUE → `{colors.navy}` SECURITY GREEN | 산업·보안·AI 관련 강조 |

## 3. 타이포그래피 규칙

딥노이드는 **Pretendard 단일 서체**만 사용합니다. 서체 통일은 브랜드 일관성의 기반입니다. 맑은 고딕을 포함한 다른 서체는 사용하지 않습니다.

> 다운로드: https://cactus.tistory.com/306

### 텍스트 계층

| 토큰 | Weight | 크기 | 범위 | 사용처 |
|---|---|---|---|---|
| `{typography.title}` | ExtraBold 800 | 36pt | 32–40pt | 슬라이드·페이지 대표 제목 |
| `{typography.subtitle}` | Bold 700 | 18pt | 14–20pt | 섹션 제목, Head Message |
| `{typography.body}` | Regular 400 | 14pt | — | 일반 본문 텍스트 |
| `{typography.table}` | Bold / Regular | 12pt | 10–14pt | 표 내부, 라벨, 다이어그램 |
| `{typography.caption}` | Regular 400 | 8pt | — | 출처, 주석, 부가 정보 |

### 타이포그래피 원칙

- **자간**: 텍스트 상자는 `normal(보통, 0)` **고정** — 좁히거나(narrow) **넓히는(wide·확장) 설정 모두 금지**. 도형 내부에 한해 `narrow(좁음)`만 상황에 따라 허용
- **행간**: 도형 내 1.1 / 텍스트 상자 1.2
- **단락 뒤 간격**: 3pt (`{spacing.paragraph-after}`)
- **한글 단어 잘림**: 허용 해제 (단어 단위 줄바꿈 적용)
- **정렬**: 왼쪽 맞춤 또는 가운데 맞춤만 허용 — 오른쪽·양쪽 맞춤 금지

## 4. 레이아웃 & 간격

### 기준 여백

모든 결과물은 아래 여백을 기준으로 합니다. 여백을 침범하는 요소는 반드시 수정합니다.

| 방향 | 값 |
|---|---|
| 상단 | `{spacing.margin-top}` = 5.3mm |
| 하단 | `{spacing.margin-bottom}` = 8.3mm |
| 좌측 | `{spacing.margin-left}` = 13.3mm |
| 우측 | `{spacing.margin-right}` = 13.3mm |

콘텐츠 블록 사이 간격: `{spacing.content-gap}` = 0.3–0.5cm

내용이 많은 페이지는 **반판 구성**(상단에 제목, 하단을 좌·우 2분할)을 활용합니다. 가독성·가중치 흐름은 **제목 → 좌 → 우** 순입니다.

|---|---|---|---|---|

도형은 두 가지 용도로만 사용합니다: **① 배경용** (레이아웃 구조) / **② 도표용** (다이어그램, 플로우차트).

| 행간 | 1.1 | 1.2 |
| 단락 뒤 간격 | 3pt | 3pt |
| 한글 단어 잘림 허용 | 해제 | 해제 |
| 텍스트 맞춤 | 중간 | — |

### 텍스트 정렬 & 줄바꿈 (AI 에이전트 필독)

도형·텍스트 상자 안의 **모든 문장**(제목·부제목·본문·라벨·도표 텍스트)은 아래 규칙을 지켜 정렬과 여백을 **통일**합니다. 텍스트를 임의로 휘갈겨 넣지 않습니다.

- **정렬 통일** — 같은 종류의 요소(카드 제목끼리, 본문끼리, 라벨끼리)는 동일한 정렬과 시작 위치를 공유합니다. 요소마다 정렬이 달라지지 않게 합니다. 정렬은 왼쪽 또는 가운데만 사용합니다.
- **줄바꿈은 의미 단위로** — 줄바꿈은 어절·구 단위의 자연스러운 위치에서 끊습니다. 단어 중간이 잘리거나(한글 단어 잘림 허용 해제 유지), 조사·접속사 한 토막만 다음 줄로 넘어가지 않게 합니다.
- **도형 밖으로 넘치지 않게** — 텍스트는 반드시 도형 경계 안에 들어와야 합니다. 넘칠 경우 ① 문장을 간결하게 다듬거나 ② 글꼴 크기를 규정 범위(`3. 타이포그래피` 참조) 안에서 줄이거나 ③ 도형 크기를 키워 해결합니다. 자간을 넓혀·좁혀 억지로 맞추지 않습니다.
- **내부 여백 통일** — 모든 도형은 상하좌우 동일한 내부 여백을 적용합니다(도형 0.1cm / 텍스트 상자 0cm). 같은 화면의 도형끼리 여백이 들쭉날쭉하지 않게 합니다.

### 기타 효과

- **하이라이트 색상**: 스포이트로 브랜드 컬러에서 흡수하여 사용
- **그림자**: blur 4pt · 거리(distance) 3pt · 각도 45° (불투명도 80%)

## 6. 내지 레이아웃 패턴 (Inner Page Patterns)

§4·§5가 결과물 전체의 골격을 정의했다면, 본 절은 **내지 본문 영역(y≈6.5cm 이하, x∈[1.33, 32.55]cm)을 채우는 5가지 재사용 가능한 콘텐츠 패턴**을 규정합니다. 사내강연 PPTX(2026-05-20) 제작에서 실측·검증된 패턴이며, 새 결과물을 만들 때 이 5종 중 1종을 골라 적용합니다. 같은 화면에는 한 가지 패턴만, **연속된 슬라이드에 같은 패턴을 두 번 이상 쓰지 않습니다.**

> **AI 에이전트 필독**: 본 절의 좌표·크기는 슬라이드 16:9 widescreen (33.87 × 19.05 cm) 기준이며, 본문 영역의 좌측 여백 `{spacing.margin-left}` = 1.33cm, 카드 사이 간격 `{spacing.content-gap}` = 0.45–0.82cm를 일관 적용합니다.

### 6.1 카드 (Card) — 기본 컴포넌트

다른 모든 패턴의 빌딩 블록. 정보 한 덩어리를 담는 최소 단위.

| 요소 | 사양 |
|---|---|
| **외곽** | 둥근 사각형(rounded-rect), 모서리 반경 ≈ `{rounded.shape}` 10px (pptxgenjs/python-pptx `adj=0.08`) |
| **테두리** | 0.75–1.0pt, 색상은 메인 2색 중 하나 (`{colors.primary}` 또는 `{colors.green}`) — 카드별로 교대 사용 |
| **채움** | `{colors.white}` 기본. 강조 카드는 메인색 채움(§6.4 KPI형) 또는 `{colors.light-blue}`/`{colors.light-green}` 배경 틴트(§6.3 비교형 AS-IS) |
| **상단 액센트 라인** | 카드 상단 0.6cm 위치, 좌·우 0.5cm 들여쓰기, 두께 2.5pt, 색상은 테두리와 동일 |
| **내부 여백** | 좌·우 0.5cm, 상단(헤더 위) 0.95cm, 하단 0.5cm |
| **헤더** | `{typography.subtitle}` 18pt Bold, `{colors.text}`(또는 메인색), 1.2 행간 |
| **본문** | `{typography.body}` 13pt Regular, `{colors.text}`, 1.3 행간 |

### 6.2 카드 그리드 (Card Grid) — 3·4·7 카드

균일한 폭의 카드를 한 줄(또는 두 줄)로 배열하여 병렬 정보를 보여주는 패턴.

| 변형 | 카드 폭 × 높이 | 간격(가로) | 행 수 | 액센트 컬러 사용 |
|---|---|---|---|---|
| **3단** | 10.07 × 9.0 cm | 0.50 cm | 1 | 가운데 카드는 GREEN, 좌·우는 BLUE (또는 그 반대) |
| **4단** | 7.46 × 9.0 cm | 0.45 cm | 1 | 카드 인덱스 i % 2 == 0 → BLUE, 1 → GREEN |
| **4×2 (7카드)** | 7.46 × 5.3 cm | 가로 0.45 / 세로 0.40 cm | 2 | 위와 동일. 8번째 칸은 `{colors.light-green}` 배경의 안내·캡션 카드로 활용 |

- 시작 좌표: `(x=1.33cm, y=6.5cm)`
- 모든 카드의 폭·높이·테두리·내부 여백을 동일하게 유지 (§5 텍스트 정렬 규칙)
- 카드별 본문 줄 수가 다를 경우 본문 줄 수에 맞춰 카드 높이를 늘리지 말고 **본문 표현을 정제**

### 6.3 비교형 (Comparison · AS-IS / TO-BE)

지금(문제)과 이후(개선)를 좌·우로 대조하는 패턴. **딥노이드 발표 자료에서 가장 자주 등장하는 패턴.**

| 영역 | 좌측 (AS-IS / Before / 함정) | 우측 (TO-BE / After / Sweet Spot) |
|---|---|---|
| **위치** | x=1.33cm, y=6.5cm | x=17.34cm, y=6.5cm |
| **크기** | 15.2 × 8.5 cm | 15.2 × 8.5 cm |
| **채움** | `{colors.light-blue}` (배경 틴트 전용 규칙 §2) | `{colors.white}` |
| **테두리** | 없음 | `{colors.green}` 1.5pt |
| **상단 라벨** | "AS-IS · 지금 우리" 등 — `{colors.navy}` 12pt Bold (비교 표현 전용색 §2) | "TO-BE · 이후" 등 — `{colors.green}` 12pt Bold |
| **헤더** | 18pt Bold, `{colors.text}` | 18pt Bold, `{colors.text}` |
| **본문** | 13pt Regular bullet 3개, `{colors.text}` | 13pt Regular bullet 3개. **마지막 핵심 bullet 1개는 `{colors.primary}` Bold** |
| **하단 캡션** | 슬라이드 하단(y=16.0cm) 가운데정렬, 14pt `{colors.muted}` — TO-BE를 요약·강조 | |

- `{colors.navy}` 사용은 §2 "비교 표현 전용" 규칙을 만족 — AS-IS 라벨에 한정
- `{colors.light-blue}` 사용은 §2 "배경 틴트 전용" 규칙을 만족 — 카드 채움 전체(텍스트·도형 채우기 아님)

### 6.4 스텝형 (Step Flow)

순차적 프로세스 또는 N단계 프레임을 좌→우 흐름으로 보여주는 패턴.

| 요소 | 사양 |
|---|---|
| **카드 수** | 3~5개 (4단 그리드 §6.2와 동일 폭/높이: 7.46 × 9.0 cm, 간격 0.45 cm) |
| **카드 컬러** | 인덱스 i % 2 == 0 → 테두리 `{colors.primary}`, 1 → `{colors.green}` |
| **상단 STEP 라벨** | y+0.5cm, 11pt Bold, 카드 색상과 동일 (예: "STEP 1") |
| **헤더** | y+1.4cm, 17pt Bold `{colors.text}`, 1.2 행간 |
| **본문** | y+3.4cm, 12pt Regular `{colors.text}`, 1.4 행간 |
| **연결 화살표** | `MSO_SHAPE.RIGHT_ARROW` 0.35 × 0.6 cm, 카드 우측 0.05cm 떨어진 위치, 채움 `{colors.primary}`, 테두리 없음 |
| **하단 메시지** | 슬라이드 하단(y=16.0cm) 가운데정렬, 15pt Bold `{colors.primary}` — 프레임의 핵심 통찰 한 줄 |

### 6.5 KPI · 메시지형 (Filled Message Cards)

핵심 메시지를 가장 강하게 전달하는 마감용 패턴. **Takehome Message 슬라이드에 권장.**

| 요소 | 사양 |
|---|---|
| **카드 수** | 3~4개 (4단 그리드 §6.2와 동일 폭/높이: 7.46 × 9.0 cm, 간격 0.45 cm) |

### 6.6 간지 챕터 글리프 — 실측 디테일

§4 "간지 페이지 구성"의 챕터 번호 글리프에 대한 실측 보완:

| 요소 | 사양 |
|---|---|
| **폰트** | Pretendard Bold (`{typography.title}`의 ExtraBold 800이 아님 — Bold 700) |

## 8. 해야 할 것 & 하지 말아야 할 것

### 해야 할 것

- 내지(Inner Page)는 **`{colors.white}` 흰색 배경**으로 고정
- 메인 색 `{colors.primary}` BLUE와 `{colors.green}` GREEN을 **둘 다 사용** — BLUE 단독 사용 금지
- 강조는 **메인 2색(BLUE·GREEN) 중심**으로, 보조색은 더할 경우 1색까지
- 표지·간지·핵심 강조에 **BLUE→GREEN 메인 그라데이션**을 적극 활용
- 보조 6색은 **정해진 역할 조건**(배경 틴트 / 비교 / 경고)에서만 사용
- 하이라이트 색상은 스포이트로 **브랜드 컬러에서 흡수**하여 사용
- `{typography.body}` 자간은 **보통(Normal)**으로 고정
- Pretendard **ExtraBold / Bold / Regular** 3가지 Weight만 사용
- 텍스트 정렬은 **왼쪽 맞춤** 또는 **가운데 맞춤** 중 선택
- 도형·텍스트 상자 안 문장은 **정렬·줄바꿈·내부 여백을 통일** — 같은 종류 요소끼리 동일하게
- 줄바꿈은 **어절·구 단위의 자연스러운 위치**에서 끊기
- 차트에 **축, 단위, 범례, 데이터 라벨** 반드시 포함
- 데이터 및 인용구의 **출처** 명확히 표기
- 내지 본문은 §6 패턴 5종(카드/카드그리드/비교형/스텝형/KPI형) 중 **한 패턴**으로 구성
- §6 패턴 선택 시 같은 패턴이 **연속된 슬라이드에 두 번 이상** 등장하지 않도록 교차 배치
- 카드 액센트 컬러는 **카드 인덱스에 따라 BLUE·GREEN을 교대**로 사용 (3카드는 가운데 GREEN, 4카드는 i%2)

### 하지 말아야 할 것

- **내지에 장식용 사이즈 바·컬러 바·사이드바·리본 사용 금지** — AI가 생성할 때 흔히 추가하는 패턴이므로 명시적으로 제거
- **제목 아래 장식용 밑줄(accent line) 사용 금지** — AI 생성 티의 대표적 패턴
- **전체 너비 가로 색상 바·사각형 띠 사용 금지** — 헤더/푸터 장식 바 포함
- **팔레트 외 색상 임의 추가 금지** — 반드시 정의된 8색 토큰 내에서 선택
- **메인 색을 BLUE 하나로만 사용 금지** — BLUE·GREEN 두 메인 색이 결과물에 함께 나타나야 함
- **한 화면에 강조색 3색 이상 사용 금지** — 주조색 + 1색 한도
- **Light Green·Light Blue를 텍스트·도형 채우기색으로 사용 금지** — 배경 틴트 전용
- **조건 없는 기능색(Navy·Red) 사용 금지** — 비교·경고 상황에서만 등장
- **텍스트 상자 자간 좁힘·넓힘 모두 금지** — 자간은 보통(0)으로 고정 (특히 **넓히지 말 것**)
- **텍스트가 도형 경계를 벗어나게 두지 말 것** — 넘치면 문구·글꼴 크기·도형 크기를 조정해 해결
- **어절을 무시한 줄바꿈 금지** — 단어 중간이 잘리거나 조사 한 토막만 넘어가는 부자연스러운 끊김 금지
- **오른쪽 정렬·양쪽 정렬 사용 금지**
- **Pretendard 외 서체 사용 금지**
- **기준 여백 침범 금지**
- **마스터의 기본 날짜·바닥글·슬라이드 번호 자리표시자 활성화 금지** — 하단 8mm 여백을 비워둠. 슬라이드 번호는 §4 우상단 헤더 영역에 한해 사용. 단, **간지의 좌하단 챕터 번호 글리프는 예외**(§4 간지 페이지 구성 참조)
- **출처 없는 데이터 사용 금지**

## 10. 활용 가이드

이 문서를 적용해 결과물을 만들거나 수정할 때의 작업 원칙입니다. AI 에이전트는 아래 순서와 규칙을 따릅니다.

1. **토큰을 단일 출처로** — 색상·서체·간격은 frontmatter에 정의된 토큰(`{colors.primary}`, `{typography.body}` 등)을 기준으로 합니다. 본문에 HEX·pt 값을 직접 적지 말고 토큰을 참조하세요.
2. **한 번에 한 페이지씩** — 슬라이드·페이지 단위로 작업합니다. 전체를 한꺼번에 바꾸지 않습니다.
3. **색을 더하기 전에 구조로 해결** — 강조가 필요하면 색상 추가보다 여백·위계·정렬을 먼저 활용합니다.
4. **팔레트 정의 ≠ 사용 권장** — 8색이 정의되어 있어도 기본은 **메인 2색(`{colors.primary}` BLUE · `{colors.green}` GREEN)** 위주입니다. BLUE 한 색만 쓰지 않습니다. 보조색은 정해진 역할 조건에서만 등장합니다.
5. **규정에 없는 것은 지어내지 않는다** — 이 문서가 정하지 않은 사항은 임의로 결정하지 말고 `11. 미정의 영역`을 따르거나 디자인 담당자에게 확인합니다.
6. **변경은 정합성 있게** — 새 규칙이 생기면 frontmatter 토큰 → 본문 표 → `8. 해야 할 것 & 하지 말아야 할 것` → `9. 체크리스트` 순으로 함께 갱신합니다.

### 템플릿 셋업 (참고 절차)

PPT 템플릿을 직접 구성·점검할 때의 작업 절차입니다 (디자인 규정이 아닌 셋업 방법).

- **색상 지정**: 보기 → 슬라이드 마스터 → 색 → 색 사용자 지정 → 팔레트 적용 후 저장
- **기본 도형·텍스트 상자**: 해당 프레임 우클릭 → "기본 도형/텍스트 상자 스타일로 설정"으로 `5. 도형 & 텍스트 상자`의 세팅값 고정
